3. What is event delegation in the context of the DOM, and why is it useful?

Answer: Event delegation is a technique in JavaScript where you add a single event listener to a parent element instead of adding multiple event listeners to individual child elements. This technique takes advantage of the event bubbling (or event propagation) mechanism, where events triggered on child elements bubble up through their ancestors in the DOM.

1. How Event Delegation Works :Event delegation is a technique in JavaScript where you add a single event listener to a parent element instead of adding multiple event listeners to individual child elements.

a. Event Bubbling : When an event occurs on an element, it first runs the handlers on that element (if any), and then the event propagates (bubbles) up to its parent elements.


b. Event Delegation : By placing an event listener on a common parent element, you can listen for events from its child elements without directly attaching listeners to each of

2. Why Event Delegation Is Useful :

a. Efficiency: Instead of adding separate event listeners to each child element, you only need one event listener on the parent. This reduces memory usage and improves performance, especially when you have many child elements.

b.Dynamic Elements: It’s very useful when dealing with dynamically added elements. If elements are added to the DOM after the initial event listeners are set up, they won’t have event listeners attached unless you use event delegation.

c.Simplified Code: Managing a single event listener instead of many makes your code cleaner and easier to maintain.
